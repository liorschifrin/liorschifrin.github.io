Project 1.1: Nametag

After configuring Prusa Slicer for the Prusa Mini, I downloaded and added the nametag file to Pursa Slicer. I had a bit of trouble editing the text and moving them until I figured out I had to double click and click and drag directly on the text. I also added a url of liorschifrin.github.io to the nametag to see if I could add the website that I will soon be making to the nametag as well. As the text was very thin I assumed that it would likely not print correctly but I wanted to try anyway, to see where the limits are in terms of small text as well. I then sliced and exported the print to a USB drive without problem. I consulted the linked documentation on color changes in Prusa Slicer but did not need to change anything as the color change was in the file. Surprisingly, the print actually succeeded. It did have the problem of filament stringing across the letters. For the large letters they were still legible, and provided a spider web look. The small letters got a little messed up and harder to read. The other problem I ran into was in Prusa Slicer the dots disappeared when slicing in the url. So it just says liorschifrin github io instead of liorschifrin.github.io, but I think it still suits its purpose.

Project 1.2: Calibration Print

I selected the double helix print for this step. I was very unsure if this print would succeed, but there were photos of it printing on the site, so I imagined it must be possible. After printing I can confirm that it was in fact possible. I find this quite impressive. As the ladders are rotating, there is a small overhang ever layer throughout the entire print. Since its so small it does make sense that it worked though. There also were bridges of over an inch long that made the rungs of the ladder, and I'm quite impressed that PLA made its way across the entire rung without seriously drooping. The print did face the problem of being very stringy, which is a farily serious asthetic problem. I'm not sure if its related to the print itself or to the printer settings. The fact that my previous print also was very stringy, and it was on the same printer with the same spool of filament, might suggest that somethign in print settings or printer settings could be affecting stringynesss. There was also a long bridge to make so I wouldn't be surprised if attempting to make a long bridge created very thin stringy layers. To attempt to solve the stringing issues, I consulted the 3D Print quality troubleshooting guide which led me to retraction settings, and to this prusa help article for specific settings to change. The two adjustments I made are:

  • Enabling "Wipe on retraction" which will first move the nozzle back to "wipe" any lasting filament after retracting before a move
  • Changing retraction length from 2.5 to 3.2, which will affect how far the filament is retracted into the extruder
There are also many other things that could be affecting stringiness such as nozzle tempereature. The prusa help article also mentioned that the filament could have gathered moisture, so I will be using a different fialment as well. It certainly seems like there are limits to 3D printing, but I don't believe I hit them. If I tried again with different settings, I believe I could produce a better result.

STL is a file format from 1987 that just has the shape of the object. It does not include any information about scale, color, materials, or print settings. Conversely, 3MF was created in 2015 for additive manufacturing and contains all relevant information in a single file. This includes texture, color, material, manufacturing data like Prusa Slicer settings, as well as designer name, description, copyright and license info, and a thumbnail. This is especially usefuly for custom supports, infils, colors, and variable layer height. The data is also stored as human readable xml and is compressed, so is often 2-3 times smaller than a similar STL file. Source: blog.prusa3d.com

Project 1.3: Open-source Assistive Devices

I printed a one handed book holder, designed by Mathis and posted on MyMiniFactory. Found on the Makers Making Change website. The goal of the device is to make it easier to hold a book open with one hand. As said in the description, "[u]sers with limited hand dexterity or use of only one hand my find this device beneficial." It seemed to work fine. I would have to read with it for a longer period of time to see how effective it really is. Though, as I have standard number of hands and hand dexterity I might not be able to judge how effective it would be for someone else. Documentation included a description and a recommendation for .2mm layer height and 10% infil. I don't think it needed anything further. I might suggest having a few different sized holes for different finger sizes for a better fit, though it should still work fine, it will just be a bit loose or tight depending on finger size.
